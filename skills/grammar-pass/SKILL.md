# Grammar Pass (Simple)

1. Work only on Markdown files explicitly named by the user.
2. Before editing each file, create a fresh backup:
   - `cp -f -- "file.md" "file.md.bak"`
3. Correct all language errors:
   - grammar
   - spelling
   - punctuation
   - small syntax mistakes
4. Keep the text stable:
   - preserve meaning, tone, and wording choices as much as possible
   - keep frontmatter keys/values unchanged unless requested
   - keep headings, links, code blocks, lists, and document structure intact
5. Do not turn this into a wording/style rewrite.
6. When done, report:
   - edited files
   - confirmation that `.bak` files were created
   - short summary of key corrections

## Guardrails

- Do not edit files implied by context only.
- Do not edit files generated by build/tools (for example `dist/`, `.astro/`, `public/raw/`).
- Stop and report if a file is not Markdown.
