# Wording Pass (Very Conservative)

1. Work only on Markdown files explicitly named by the user.
2. Before editing each file, create a fresh backup:
   - `cp -f -- "file.md" "file.md.bak"`
3. Apply only minimal wording fixes:
   - clarify awkward phrasing only when needed
   - reduce obvious repetition only when it hurts readability
   - keep edits as small as possible
4. Be highly conservative:
   - preserve author voice and rhythm
   - keep claims, tone, and scope unchanged
   - keep frontmatter, headings, links, code blocks, lists, and structure intact
   - do not add/remove sections
5. Correct grammar/punctuation only when needed to keep the wording edit clean.
6. When done, report:
   - edited files
   - confirmation that `.bak` files were created
   - short summary of meaningful wording changes

## Guardrails

- Do not edit files implied by context only.
- Do not edit files generated by build/tools (for example `dist/`, `.astro/`, `public/raw/`).
- Stop and report if a file is not Markdown.
