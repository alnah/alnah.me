I build backend systems and developer tools in Go.

## Projects

- [alnah.io](https://alnah.io): My technical blog about Go development.
- [go-md2pdf](https://github.com/alnah/go-md2pdf) (50+ stars): Library and CLI for Markdown to PDF. Cover pages, auto table of contents, watermarks, signatures, CSS themes, and custom assets support. Parallel batch processing via headless Chrome. No LaTeX. No complexity.
- [go-transcript](https://github.com/alnah/go-transcript): CLI to transcribe meetings, lectures, and brainstorms into structured Markdown. Uses OpenAI for transcription and DeepSeek/OpenAI for AI-powered formatting.

[GitHub](https://github.com/alnah) · [LinkedIn](https://linkedin.com/in/alnah)

## Background

Trilingual in French, English, and Portuguese. I taught French for years: breaking down complex rules into clear steps, understanding why a learner gets stuck, adapting methods to how they think.

## Learning Path

I first touched Python 2 many years ago, then stepped away from programming. In 2024, I came back through [Automate the Boring Stuff with Python](https://automatetheboringstuff.com/), [CS50](https://cs50.harvard.edu/x/), and [The Odin Project](https://www.theodinproject.com/). In 2025, I completed [Boot.dev](https://www.boot.dev/)'s backend path and started building with Go.

Books that shaped my approach:

- [Learning Go](https://www.oreilly.com/library/view/learning-go-2nd/9781098139285/) by Jon Bodner
- [Learn Go with Tests](https://quii.gitbook.io/learn-go-with-tests) by Chris James
- [Dive Into Design Patterns](https://refactoring.guru/design-patterns/book) by Alexander Shvets
- [A Philosophy of Software Design](https://web.stanford.edu/~ouster/cgi-bin/book.php) by John Ousterhout
